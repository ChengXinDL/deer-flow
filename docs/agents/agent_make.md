**步骤 2：创建 `prompt.py` 文件**

## 3. 架构设计经验

### 3.1 模块结构

**推荐结构**：
```
agent_name/
├── __init__.py          # 模块导出
├── agent.py             # 主代理实现
└── prompt.py            # 系统提示模板
```

**优势**：
- 职责分离，代码结构清晰
- 易于维护和扩展
- 符合项目整体架构风格

### 3.2 中间件体系

**核心中间件**：
1. `ThreadDataMiddleware` - 线程数据管理
2. `UploadsMiddleware` - 文件上传处理
3. `DanglingToolCallMiddleware` - 工具调用补丁
4. `SummarizationMiddleware` - 对话摘要
5. `TodoListMiddleware` - 任务列表管理
6. `MemoryMiddleware` - 记忆管理
7. `ClarificationMiddleware` - 澄清请求

**中间件顺序**：
- 数据管理中间件 → 处理中间件 → 功能中间件 → 交互中间件

### 3.3 配置管理

**运行时配置**：
- 模型名称解析和验证
- 数据库连接配置
- 构建 ID 参数验证
- 思考模式支持
- LangSmith 追踪元数据

**配置验证**：
- 必须参数检查
- 默认值处理
- 错误提示和日志记录
