# 项目执行规则

## 1. 架构设计规则

### 1.1 模块化结构
- **标准结构**：采用 `__init__.py` + `agent.py` + `prompt.py` 的标准模块结构
- **职责分离**：明确模块职责，代码结构清晰，易于维护和扩展
- **一致性**：与项目其他模块保持一致的架构风格，减少认知负担

### 1.2 中间件体系
- **分层设计**：数据管理 → 处理 → 功能 → 交互的中间件执行顺序
- **按需启用**：基于配置启用中间件，避免不必要的性能开销
- **扩展性**：设计可插拔的中间件接口，便于功能扩展

### 1.3 配置管理
- **运行时配置**：支持动态配置解析和验证
- **参数验证**：严格的配置参数检查，提供清晰的错误提示
- **默认值处理**：为可选参数提供合理的默认值

## 2. 代码质量规则

### 2.1 代码风格
- **命名规范**：函数名 `snake_case`，类名 `PascalCase`，变量名 `snake_case`，常量 `UPPER_SNAKE_CASE`
- **代码组织**：按功能分组，逻辑分离，保持代码结构清晰
- **注释规范**：关键逻辑必须有注释，复杂函数必须有文档字符串

### 2.2 文档规范
- **文档字符串**：每个函数必须包含功能说明、参数描述、返回值和异常处理
- **类型提示**：使用 `typing` 模块提供类型注解，提高代码可读性
- **文档更新**：代码变更时同步更新相关文档

### 2.3 错误处理
- **异常处理**：使用适当的异常类型，提供明确的错误消息
- **错误预防**：输入验证、边界检查、配置验证
- **日志记录**：关键操作和错误必须记录日志，便于问题排查

## 3. 开发流程规则

### 3.1 分析与规划
- **分析先行**：先分析现有代码结构和模式，再进行开发或重构
- **任务分解**：将复杂任务分解为可管理的小步骤，逐步推进
- **优先级排序**：根据重要性和依赖关系确定任务优先级

### 3.2 实施与验证
- **分步骤实施**：按照规划的步骤逐步实施，确保每步都能正常工作
- **持续验证**：每完成一个步骤就进行验证，确保代码正确性
- **版本控制**：使用版本控制工具管理代码变更，保留变更历史

### 3.3 测试与质量保证
- **多层次测试**：语法检查 → 单元测试 → 集成测试 → 性能测试
- **测试覆盖**：确保关键功能和边界情况都有测试覆盖
- **代码质量**：使用代码质量工具检查代码风格和潜在问题

## 4. 性能优化规则

### 4.1 资源管理
- **内存管理**：合理的内存使用策略，避免内存泄漏
- **缓存策略**：对频繁访问的数据使用适当的缓存机制
- **资源释放**：及时释放不再使用的资源

### 4.2 查询优化
- **索引使用**：合理使用数据库索引，提高查询速度
- **查询计划**：分析和优化查询计划，减少不必要的操作
- **批量操作**：使用批量操作减少网络往返和数据库负载

### 4.3 执行效率
- **中间件优化**：优化中间件执行顺序，避免不必要的中间件
- **并行处理**：对于独立任务，使用并行处理提高效率
- **延迟加载**：按需加载资源，避免启动时加载所有资源

## 9. Python 版本兼容性规则

### 9.1 版本要求
- **项目版本**：项目使用 Python 3.12.x
- **系统版本**：系统默认环境可能是 Python 3.10.x
- **版本差异**：注意不同 Python 版本之间的语法和库差异

### 9.2 兼容性处理
- **装饰器兼容性**：`override` 装饰器在 Python 3.11 中引入，Python 3.10 不可用
- **类型提示**：使用 `typing` 模块的类型提示，避免使用仅在更高版本中可用的特性
- **条件导入**：对于版本相关的特性，使用条件导入或运行时检查

### 9.3 代码规范
- **避免版本特定特性**：不使用仅在特定 Python 版本中可用的特性
- **兼容性测试**：在目标 Python 版本中进行测试，确保代码兼容性
- **版本声明**：在项目文档中明确声明支持的 Python 版本

## 10. 包管理工具使用规则

### 10.1 包管理工具
- **主要工具**：项目使用 `uv` 作为包管理工具
- **避免混用**：不要混用 `pip` 和 `uv`，统一使用 `uv`
- **依赖安装**：使用 `uv add package_name` 安装依赖

### 10.2 依赖管理
- **依赖声明**：在 `pyproject.toml` 中声明项目依赖
- **锁定文件**：使用 `uv.lock` 文件确保依赖版本一致性
- **依赖更新**：定期更新依赖，但需要测试兼容性

### 10.3 虚拟环境管理
- **虚拟环境路径**：`g:\AI_works\10_MAGICflow\backend\.venv`
- **激活虚拟环境**：使用 `& .venv\Scripts\Activate.ps1` 激活虚拟环境
- **环境隔离**：必须使用项目的虚拟环境，不要使用系统默认环境

### 10.4 命令执行规范
- **Python 执行**：使用 `.venv\Scripts\python` 执行 Python 脚本
- **包安装**：使用 `uv add package_name` 安装包
- **包列表**：使用 `uv pip list` 查看已安装的包

## 11. 编码问题处理规则

### 11.1 文件编码
- **标准编码**：所有源代码文件必须使用 UTF-8 编码
- **编码声明**：在文件开头声明编码（可选，但推荐）
- **编码检查**：遇到编码问题时，首先检查文件编码

### 11.2 Unicode 处理
- **乱码识别**：识别并移除文件中的乱码字符
- **字符范围**：使用 Unicode 字符范围检查中文等非 ASCII 字符
- **字符串处理**：正确处理包含非 ASCII 字符的字符串

### 11.3 编码错误解决
- **错误识别**：识别 `SyntaxError: (unicode error)` 等编码错误
- **文件检查**：使用文本编辑器检查文件编码设置
- **重新保存**：以 UTF-8 编码重新保存文件

### 12.1 Agent 类型选择
