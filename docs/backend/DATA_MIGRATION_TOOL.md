# 数据迁移工具文档

## 概述

数据迁移工具用于将数据从A数据库（源数据库）迁移到B数据库（目标数据库），基于guid字段进行数据筛选和迁移。

## 功能特性

1. **基于guid的数据提取**：从A数据库所有表中提取指定guid的数据
2. **表结构验证与同步**：自动验证B数据库表结构，必要时创建与A数据库相同的表结构
3. **数据覆盖策略**：如果B数据库中已存在相同guid的数据，进行覆盖更新
4. **完整性验证**：迁移完成后验证数据完整性
5. **详细日志记录**：记录迁移过程的详细信息到日志文件
6. **错误处理**：遇到错误时停止迁移并记录错误信息

## 使用方法

### 基本用法

```bash

```

### 参数说明

- `guid`：必需参数，用于筛选数据的guid值

### 示例

```bash
uv run python backend/scripts/migrate_data_by_guid.py "335834679343"
```

## 迁移流程

### 1. 初始化阶段
- 解析命令行参数获取guid值
- 初始化日志记录器
- 连接到源数据库和目标数据库
- 验证数据库连接

### 2. 表结构验证阶段
- 获取源数据库中所有表名
- 对每个表进行表结构验证：
  - 检查目标数据库中是否存在该表
  - 如果不存在，从源数据库获取表结构并在目标数据库中创建
  - 如果存在，比较表结构是否一致
  - 如果结构不一致，记录警告并停止迁移

### 3. 数据提取阶段
- 对每个包含guid字段的表：
  - 查询源数据库中guid等于指定值的所有记录
  - 记录查询到的记录数量

### 4. 数据迁移阶段
- 对每个查询到的记录：
  - 检查目标数据库中是否存在相同guid的记录
  - 如果存在，执行UPDATE操作覆盖数据
  - 如果不存在，执行INSERT操作插入新数据
  - 记录每次操作的结果

### 5. 完整性验证阶段
- 对每个迁移的表：
  - 比较源数据库和目标数据库中的记录数量
  - 验证关键字段值是否一致
  - 记录验证结果

### 6. 清理阶段
- 关闭数据库连接
- 生成迁移报告
- 记录迁移完成时间和总耗时

## 日志记录

### 日志文件位置
- 日志文件保存在：`backend/logs/migrate_data_by_guid_<guid>_<timestamp>.log`

### 日志级别
- **INFO**：记录正常操作流程
- **WARNING**：记录警告信息（如表结构不一致）
- **ERROR**：记录错误信息（如连接失败、迁移失败）
- **DEBUG**：记录详细的调试信息

### 日志内容
- 迁移开始时间
- 数据库连接信息
- 表结构验证结果
- 数据提取详情
- 数据迁移详情
- 完整性验证结果
- 迁移完成时间和总耗时

## 错误处理

### 错误类型

1. **数据库连接错误**
   - 无法连接到源数据库或目标数据库
   - 处理方式：停止迁移，记录错误信息

2. **表结构不一致错误**
   - 目标数据库表结构与源数据库不一致
   - 处理方式：停止迁移，记录错误信息

3. **数据迁移错误**
   - INSERT或UPDATE操作失败
   - 处理方式：停止迁移，记录错误信息

4. **完整性验证错误**
   - 验证发现数据不一致
   - 处理方式：记录警告，继续其他表的验证

### 错误恢复
- 脚本在遇到错误时会停止执行
- 日志文件会记录详细的错误信息
- 用户可以根据日志信息定位问题并修复后重新执行

## 注意事项

1. **数据备份**：在执行迁移前，建议备份目标数据库
2. **权限要求**：执行脚本的用户需要有源数据库的读取权限和目标数据库的读写权限
3. **网络连接**：确保能够访问两个数据库服务器
4. **表结构一致性**：如果表结构不一致，脚本会停止迁移，需要手动处理
5. **guid字段**：只有包含guid字段的表才会被处理
6. **事务处理**：每个表的迁移在一个事务中执行，确保数据一致性

## 性能考虑

1. **批量操作**：对于大量数据，考虑使用批量插入和更新
2. **索引优化**：确保guid字段有索引以提高查询性能
3. **事务大小**：对于大表，考虑分批处理以避免事务过大
4. **网络延迟**：跨网络迁移时考虑网络延迟对性能的影响

## 安全考虑

1. **密码安全**：数据库密码存储在环境变量中，不在代码中硬编码
2. **SQL注入**：使用参数化查询防止SQL注入
3. **权限控制**：使用最小权限原则配置数据库用户
4. **日志安全**：日志中不记录敏感信息（如密码）
